# PostgreSQL (CloudNativePG)

Orders database cluster managed by the [CloudNativePG](https://cloudnative-pg.io/) operator.

| Resource | Value |
|----------|-------|
| Cluster CR | `eurotransit-orders-db` |
| Namespace | `eurotransit` |
| Database | `ordersdb` |
| Application user | `app` (owner) |
| K8s secret | `eurotransit-orders-db-app` (generated by the operator) |
| Read/write service | `eurotransit-orders-db-rw.eurotransit.svc:5432` |
| Read-only service | `eurotransit-orders-db-ro.eurotransit.svc:5432` |

## GitOps

Synced by the `eurotransit-data` Argo CD Application (`apps/data-infrastructure.yaml`), a child of the root app-of-apps.

## Local bootstrap (k3d)

After the cluster and CNPG operator are up:

```bash
just deploy-postgres
kubectl -n eurotransit get cluster eurotransit-orders-db
kubectl -n eurotransit get secret eurotransit-orders-db-app
```

Schema migrations are owned by the Orders service (Flyway) in `eurotransit-app`:
`backend/orders-service/src/main/resources/db/migration/`.
