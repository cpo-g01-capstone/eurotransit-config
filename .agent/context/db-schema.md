# Orders database schema

Operator: CloudNativePG  
Cluster CR name: `eurotransit-orders-db` (`postgres/eurotransit-orders-db.yaml`)  
Read/write service: `eurotransit-orders-db-rw.eurotransit.svc:5432`  
Database name: `ordersdb`  
Application user: `app`  
Application secret: `eurotransit-orders-db-app` (generated by CloudNativePG)

Migrations: Flyway in `eurotransit-app` → `backend/orders-service/src/main/resources/db/migration/`

## Tables

### `order_status` (enum)
`DRAFT` | `RESERVED` | `PAID` | `CONFIRMED` | `FAILED`

### `orders`
| Column | Type | Notes |
|--------|------|-------|
| id | UUID PK | client-supplied; idempotency key for `order-placed` |
| status | order_status | lifecycle state |
| created_at | TIMESTAMPTZ | default `NOW()` |
| updated_at | TIMESTAMPTZ | default `NOW()` |

### `processed_events`
| Column | Type | Notes |
|--------|------|-------|
| event_id | VARCHAR(512) PK | dedup key: `{orderId}:{eventType}` (Kafka consumers) |
| processed_at | TIMESTAMPTZ | default `NOW()` |

### `idempotency_records`
| Column | Type | Notes |
|--------|------|-------|
| idempotency_key | VARCHAR(255) PK | HTTP `Idempotency-Key` header / sync API |
| response_payload | JSONB | cached response for duplicate requests |
| created_at | TIMESTAMPTZ | default `NOW()` |

Initial migration: `V1__init_orders_schema.sql` (EM-16).
